### Comment 5 ‚Äî github-actions (2026-03-16T15:25:05Z)

The bug: install stdout/stderr is suppressed so we can't see failures, and if `ollama` ends up in `/usr/local/bin` but PATH isn't updated, the `Popen(["ollama", ...])` fails. Fix:

Fixed and pushed. Two root causes:

1. **Silent failure** ‚Äî install errors were swallowed (`stdout=DEVNULL`). Now we `capture_output=True`, check `returncode`, and raise with the stderr tail if it fails.
2. **Binary not in PATH** ‚Äî after `curl | sh`, the shell's PATH isn't refreshed for the Python process. Now we do `shutil.which("ollama")` first, then fall back to `/usr/local/bin/ollama` (where the install script puts it), and raise a clear `FileNotFoundError` with install output if neither works.
