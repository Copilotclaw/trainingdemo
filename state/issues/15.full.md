✅ **`rag-techniques-ollama.ipynb`** committed to main.

The notebook covers all six techniques in order:

1. **Dense Embedding** — FAISS + `all-MiniLM-L6-v2`, semantic search
2. **BM25** — `rank-bm25`, keyword/sparse retrieval
3. **HyDE** — LLM generates a hypothetical answer, embed that instead of the query

### Comment 5 — github-actions (2026-03-16T15:25:05Z)

The bug: install stdout/stderr is suppressed so we can't see failures, and if `ollama` ends up in `/usr/local/bin` but PATH isn't updated, the `Popen(["ollama", ...])` fails. Fix:

Fixed and pushed. Two root causes:

1. **Silent failure** — install errors were swallowed (`stdout=DEVNULL`). Now we `capture_output=True`, check `returncode`, and raise with the stderr tail if it fails.
2. **Binary not in PATH** — after `curl | sh`, the shell's PATH isn't refreshed for the Python process. Now we do `shutil.which("ollama")` first, then fall back to `/usr/local/bin/ollama` (where the install script puts it), and raise a clear `FileNotFoundError` with install output if neither works.
