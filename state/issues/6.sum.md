

### 2026-03-09

The human RAG exercise concept was implemented as a 7-step interactive notebook called `human-rag-exercise.ipynb`. The notebook simulates a paper-based research assistant scenario with 20 pet-related articles and 4 customer inquiries to demonstrate RAG challenges, including a hallucination trap. A mapping table connects the manual process to actual RAG components (corpus, retrieval, embeddings, etc.). The exercise emphasizes the difficulty of information retrieval and grounding issues when scaling from 20 to thousands of documents. The notebook was added as the first item in the README as the recommended starting point before the simple RAG demo.

### 2026-03-09

The MCP (Model Context Protocol) notebook idea was abandoned in favor of modern software agent patterns. An `simple-rag-demo-ollama.ipynb` was created as the Ollama local version of the RAG demo, featuring real LLM generation with `ollama.chat()`, model selection options, and hallucination testing. The notebook supports both local and Colab environments with automatic Ollama installation and includes streaming token output. A bonus Exercise 4 demonstrates swapping SentenceTransformers for Ollama's `nomic-embed-text` to create a fully local stack.

### 2026-03-09

The question about Ollama compatibility with free Colab was answered affirmatively, confirming the existing notebook implementation already handles this setup. The notebook includes auto-installation of Ollama and proper GPU utilization on T4 hardware available in free Colab sessions. The main consideration identified was potentially increasing the initialization wait time from 3 to 10 seconds to ensure Ollama properly starts before model operations begin. No new artifacts were created for this specific query, as the functionality was already implemented in `simple-rag-demo-ollama.ipynb`.
