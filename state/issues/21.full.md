### Comment 3 — github-actions (2026-03-17T15:32:55Z)

The issue is clear — the GPG key needs to be **dearmored** (converted from ASCII PEM to binary). The raw download without `gpg --dearmor` causes the "not signed" error. Let me fix Cell 2.

Fixed. 🦃

**Root cause**: `wget -qO /path/key.gpg URL` saves the raw ASCII-armored PEM file. APT expects binary keyring format — without `gpg --dearmor`, apt can't verify the signature → `NO_PUBKEY` error.

**Fix** (one pipe added):
```bash
wget -qO - https://debian.neo4j.com/neotechnology.gpg.key \
    | gpg --dearmor -o /etc/apt/keyrings/neo4j.gpg
```

Pushed to main — re-run the Colab cell and the GPG error should be gone.

### Comment 5 — github-actions (2026-03-17T15:39:50Z)

The `gpg: cannot open '/dev/tty'` error is because gpg tries to open a TTY for interactive prompts — Colab's shell doesn't have one. Fix: add `--batch --no-tty`.

Good, the patch is in. Let me commit and push:

Fixed ✅

**Root cause**: Colab runs cells in a non-interactive shell with no `/dev/tty`. GPG tries to open it for passphrase prompts → fails → key never gets written → `NO_PUBKEY`.

**Fix** — added `--batch --no-tty` to the gpg call:
```bash
wget -qO - https://debian.neo4j.com/neotechnology.gpg.key \
    | gpg --batch --no-tty --dearmor -o /etc/apt/keyrings/neo4j.gpg
```

Pushed to main. Re-run Cell 2 and the GPG error should be gone.
