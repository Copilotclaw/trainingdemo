Installing Ollama on Google Colab

Why your install silently fails — and the one-line fix

🔧 zstd fix ☁️ Google Colab 🦙 Ollama 🐍 Python
🦙 What is Ollama?

Ollama is an open-source tool that lets you download and run large language models (like llama3.2, mistral, phi3) entirely on your own machine — no API key, no internet dependency, no usage fees.

On Google Colab you get a free GPU/CPU environment, which makes it tempting to run Ollama there for demos and experiments. The official install is a single curl | sh command. Simple — except when it isn't.

💥 The Problem
⚠️
The Ollama install script exits 0 (success) — but leaves no binary behind. Your Python code then crashes with FileNotFoundError: [Errno 2] No such file or directory: 'ollama'

Root Cause: Missing zstd

Ollama packages its Linux binary using the Zstandard (zstd) compression format — a fast, high-ratio compressor from Facebook/Meta.

The install script (https://ollama.ai/install.sh) downloads an archive and uses zstd to decompress it before placing the binary in /usr/local/bin/.

Google Colab's Ubuntu environment does not ship with zstd by default. The decompression step silently fails, the binary is never created, and the install script still exits 0 — giving you no warning at all.

🔍 The Error Chain

1
curl | sh runs the install script Script downloads ollama-linux-amd64.tar.zst and tries to extract it.
2
zstd not found — extraction fails silently The script continues anyway, exits 0. Nothing in Python catches this.
3
No binary at /usr/local/bin/ollama The file was never written to disk.
4
Popen(["ollama", "serve"]) raises FileNotFoundError Python can't exec a binary that doesn't exist.
✅ The Fix

Install zstd with a single apt-get command before running the Ollama install script. That's it.

❌ Broken (no zstd)
import subprocess # Install Ollama — no zstd, silently fails subprocess.run( "curl -fsSL https://ollama.ai/install.sh | sh", shell=True ) # 💥 FileNotFoundError! subprocess.Popen(["ollama", "serve"])
✅ Fixed (with zstd)
import subprocess # Step 1: install zstd FIRST subprocess.run( "apt-get install -y zstd", shell=True, check=True ) # Step 2: now Ollama installs correctly subprocess.run( "curl -fsSL https://ollama.ai/install.sh | sh", shell=True, check=True )
💡
One line saves you 20 minutes of debugging. apt-get install -y zstd takes about 2 seconds on Colab and prevents the entire silent-fail cascade.
📋 Complete Colab Setup Cell

Drop this into the first code cell of any notebook that uses Ollama. It auto-detects Colab, installs everything, starts the server, and pulls the model.

import subprocess, time, os, shutil def is_colab(): try: import google.colab; return True except ImportError: return False if is_colab(): print("Colab detected — installing Ollama...") # ✅ zstd is required by the Ollama install script — install it first! subprocess.run("apt-get install -y zstd", shell=True, check=True) # Install Ollama (now zstd is available, extraction succeeds) result = subprocess.run( "curl -fsSL https://ollama.ai/install.sh | sh", shell=True, capture_output=True, text=True ) if result.returncode != 0: print("⚠️ Install stderr:", result.stderr[-1000:]) raise RuntimeError("Ollama install failed") print("Install complete.") # Locate binary (install.sh puts it in /usr/local/bin) ollama_bin = shutil.which("ollama") or "/usr/local/bin/ollama" if not os.path.exists(ollama_bin): raise FileNotFoundError( f"Ollama binary not found at {ollama_bin}. stdout: {result.stdout[-500:]}" ) print(f"Ollama binary: {ollama_bin}") # Start server in the background subprocess.Popen( [ollama_bin, "serve"], stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL ) time.sleep(5) # wait for server to start # Pull the model subprocess.run([ollama_bin, "pull", "llama3.2"], check=True) print("✅ Ollama ready") else: print("Local run — assuming Ollama is already running with llama3.2 pulled")

💡 Key Details

capture_output=True — captures stdout/stderr so you can see what went wrong if the install fails.

shutil.which("ollama") — finds the binary using PATH lookup first, then falls back to the known install location.

time.sleep(5) — the server needs a few seconds to bind its port after starting.

is_colab() guard — skips the whole setup when running locally where Ollama is already installed.

🗜️ Why zstd?

Zstandard — Fast Modern Compression

Zstandard (zstd) is a fast, lossless compression algorithm developed by Facebook/Meta. Released in 2016, it offers significantly better speed than gzip/bzip2 at comparable compression ratios.

Ollama chose .tar.zst for its release archives to keep download sizes small while allowing fast decompression on target machines. It's standard on modern Linux desktops — but Colab's Ubuntu environment is minimal and doesn't include it by default.

📦 Why Colab Doesn't Have It

Google Colab runs on minimal Ubuntu images optimised for ML workloads (CUDA, Python, Jupyter). Tools that come pre-installed are carefully curated for size. zstd, while common on modern desktop Linux distros (Ubuntu 20.04+, Arch, Fedora), is not in the default Colab image.

Installing it takes ~2 seconds and costs nothing. One apt-get install -y zstd and you're done.

🛠️ Troubleshooting
Symptom Likely Cause Fix
FileNotFoundError: 'ollama' zstd missing — binary was never created Add apt-get install -y zstd before the install script
Install exits 0 but binary not in PATH PATH not refreshed in Python process after install Use shutil.which("ollama") or "/usr/local/bin/ollama"
Connection refused on ollama API calls Server started but not ready yet Increase time.sleep() to 8–10 seconds
Model pull fails or times out Colab network instability or model too large Retry the pull cell; use smaller model like phi3 or llama3.2:1b
Colab runtime disconnects during model pull Large model (7B+) takes too long Stick to 3B models on free Colab; pull with a timeout wrapper
Install stderr shows zstd: command not found Setup cell ran in wrong order Re-run the cell that installs zstd first, then re-run install
