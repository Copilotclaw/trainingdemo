How it works

The Full RAG Pipeline

RAG solves the "LLMs don't know your documents" problem. You build a searchable index of your knowledge, retrieve the relevant bits for each question, and hand them to the LLM as context.

1

Build the Knowledge Base

You start with a list of documents — anything from a few sentences to thousands of paragraphs. These are the facts your RAG system will search through.

python documents = [ "RAG stands for Retrieval-Augmented Generation. It combines search with LLMs.", "FAISS is a library by Meta for efficient similarity search on dense vectors.", "Local RAG runs entirely on your machine — no API calls, no data leaving your system.", "Ollama lets you run models like Llama 3 or Mistral locally with one command.", # ... 4 more documents ] print(f'📚 {len(documents)} documents loaded')
💡 Real world: Documents would come from PDFs, markdown files, code repos, or database exports — split into chunks of ~300–500 tokens each.
2

Embed the Documents

A sentence-transformer model converts each document into a dense vector — a list of numbers that captures its meaning. Similar documents end up with numerically similar vectors.

python model = SentenceTransformer('all-MiniLM-L6-v2') # ~80MB, fast, great quality embeddings = model.encode(documents, show_progress_bar=True) embeddings = np.array(embeddings).astype('float32') print(embeddings.shape) # → (8, 384) — 8 docs, 384 numbers each
💡 Why 384? The all-MiniLM-L6-v2 model produces 384-dimensional vectors. That's 384 floating-point numbers that together capture the semantic meaning of a sentence.
3

Build the FAISS Index

FAISS (Facebook AI Similarity Search) is a hyper-optimized library for finding the nearest vectors in a large set. We dump all our document embeddings into it.

python dimension = embeddings.shape[1] # 384 index = faiss.IndexFlatL2(dimension) # L2 = Euclidean distance index.add(embeddings) print(f'🗂️ FAISS index built with {index.ntotal} vectors')
💡 IndexFlatL2 does exact brute-force search — fine for small datasets. For millions of documents you'd use IndexIVFFlat (approximate, but 100× faster).
4

Query the Index

When a user asks a question, we embed the question using the same model, then search FAISS for the top-k most similar document vectors.

python def search(query: str, top_k: int = 3) -> list[str]: query_vec = model.encode([query]).astype('float32') distances, indices = index.search(query_vec, top_k) return [(documents[i], distances[0][j]) for j, i in enumerate(indices[0])] # Example query for doc, score in search("How does local RAG work without internet?"): print(f'[{score:.4f}] {doc}')
5

Build the Prompt (Generation)

The retrieved documents become context injected into the LLM prompt. The LLM answers using both its training knowledge AND the retrieved facts.

python def build_prompt(query: str) -> str: context_docs = search(query, top_k=3) context = '\n'.join([f'- {doc}' for doc, _ in context_docs]) return f"""You are a helpful assistant. Context: {context} Question: {query} Answer:"""
💡 In this notebook we just show the formatted prompt. In real RAG you'd send this to Ollama, OpenAI, or any LLM API to get an actual grounded answer back.
Glossary

Key Concepts

📐

Embeddings

Dense numerical vectors that represent text in a high-dimensional space. Similar text → numerically close vectors. The core magic of semantic search.

🗂️

Vector Index

A data structure optimized for finding the nearest vectors. FAISS can search millions of vectors in milliseconds using Euclidean or cosine distance.

✂️

Chunking

Splitting long documents into smaller pieces (300–500 tokens) before embedding. Smaller chunks = more precise retrieval. Too small = lost context.

🎯

Top-k Retrieval

Finding the k most similar documents to a query. k=3 is common — enough context without overwhelming the LLM's context window.

📏

L2 Distance

Euclidean distance between vectors. Smaller = more similar. IndexFlatL2 computes exact distances — perfect for small datasets.

🏠

Local RAG

Running the entire pipeline on your machine: local embeddings (SentenceTransformers) + local LLM (Ollama). Zero data leaves your system. Privacy by design.
