🤔 What is RAG?

The Problem with Plain LLMs

LLMs are trained on static data — they don't know about your documents, your codebase, or anything that happened after their training cutoff. RAG solves this by fetching relevant information at query time and handing it to the LLM as context.

The RAG Idea

Instead of fine-tuning (expensive, slow), you retrieve the right chunks from your knowledge base and augment the prompt. The LLM can then answer with up-to-date, specific information — without ever being retrained.

⚙️ The 5-Step Pipeline
1

📚 Load Documents

Start with raw text — sentences, paragraphs, or chunks from your knowledge base. These become the facts your system can retrieve.

2

🔢 Embed

A sentence-transformer model converts each text chunk into a 384-dimensional vector (a list of numbers). Similar meanings → similar vectors.

3

🗂️ Index (FAISS)

All vectors go into a FAISS index — a lightning-fast search engine for vectors. It enables nearest-neighbor lookup in milliseconds.

4

🔎 Query

Embed the user's question with the same model, then ask FAISS: which stored vectors are closest? Those documents are your context.

5

💬 Generate

Stuff the retrieved chunks + the user question into an LLM prompt. The model answers using real, relevant context — not hallucination.

🔢 Embeddings Visualized

Why Vectors Work

The magic: "Local RAG" and "runs on your machine" encode to nearby points in 384D space, even though the words are different. The distance between vectors = semantic distance between meanings. FAISS searches this space in O(log n) time.

💻 Code Walkthrough

Step 2 — Embed Documents

Load a pretrained model (~80MB) and embed all documents in one batch call:

from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer('all-MiniLM-L6-v2')  # tiny but mighty
embeddings = model.encode(documents).astype('float32')
# shape → (8, 384) — 8 docs, 384 dimensions each

Step 3 — Build FAISS Index

Create a flat L2 index and add all embeddings in one shot:

import faiss

index = faiss.IndexFlatL2(384)  # dimension must match embedding size
index.add(embeddings)           # O(1) per document
# Now index.ntotal == 8

Step 4 — Search

Turn a user question into a vector, find the top-3 closest documents:

def search(query: str, top_k: int = 3):
    query_vec = model.encode([query]).astype('float32')
    distances, indices = index.search(query_vec, top_k)
    return [(documents[i], distances[0][j]) for j, i in enumerate(indices[0])]

results = search("How does local RAG work without internet?")
# → [("Local RAG runs entirely on your machine...", 0.14), ...]

Step 5 — Build Prompt

Insert the retrieved context into the LLM prompt template:

def build_prompt(query: str) -> str:
    context = '\n'.join([f'- {doc}' for doc, _ in search(query)])
    return f"""Context:
{context}

Question: {query}

Answer:"""

# In a real system, pass this to ollama.chat() or openai.chat.completions.create()
�� Key Concepts
📐 L2 Distance
Euclidean distance between vectors. Lower = more similar.
✂️ Chunking
Split large docs into smaller pieces before embedding.
⚡ FAISS
Meta's library for billion-scale vector search.
🤗 Sentence Transformers
Pretrained models that encode text to dense vectors.
🏠 Local RAG
Everything on your machine. No API, no data leaks.
🦙 Ollama
Run Llama 3 / Mistral locally for the generation step.
